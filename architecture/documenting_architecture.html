Documenting Architecture

Guidelines

Alwyas use arrow.

Avoid using lines.

Avoid using bidirectional arrows, instead use two arrows

Use the solid arrow to represent request and if you want to represent the response flow as well, use dotted arrow

Change the direction of arrow to database based on read or write

Use solid lines for synchronous calls and larged dotted lines for asynchronous calls. This helps to understand the coupling level between the components.
