What is NFR?

NFR is a requirement that specifies criteria that can be used to judge the operation of a system, rather than specific behavior.

Functional requirements define what a system is supposed to do and non-functionl requirements define how a system is supposed to be.

7 important NFRs

  • Scalability
  • Performance
  • Testability
  • Security
  • Extensibility
  • Observability
  • Maintainability

Why 7 NRFs

  • Too important to ignore.
  • Easily overseen.
  • Too costly to oversee.
  • Help to build our credibility.
  • Ignoring may create a blackspot in our career.

Scalability

Scalability Problems:
If considered too early u end up YAGNI, too late wast money in resources.
MVP suddenly blows up into real project.

What is Scalability:
Scalability means the capacity to be changed in size or scale.
Scalability is the most important NFR, which definitely needs an architect to be present in the project.

Scalability Tips
Automation is the key, Human intervention kills scalability.
Use Content Delivery Network / Load Balancer / Traffic Manager / Geo Replication
Enable Auto scaling.
Think about messaging patterns.
Choose proper DB based on requriements and storage type. NoSQL Vs SQL Options.
Enable cache and use them extensively. Browser cache / Client-side Cache / Server-side Cache / Data-base cache.

Performance

Performance Problems
If your application is not perfromant you are out of business.
People nowadays wants everything now. They don't want to wait.

What is Performance?
Performance is an indication of the responsiveness of a system to execute any action within a given time interval.

Performance Tips
Define a metric for during the intial phases of your design and get it signed from the stakeholders.
Measure KPI's
    Average Response Time
    Error Rate
    Request Rate
    Application Availability
    Mean time between failres
    Mean time to recover/ repair
Always choose push communcation instead of pull communcation.
Choose Horizantal scaling as much as possible.
Realizing speed is relative, a good progress bar will make your app like a quick one.

Testability

Testability Problems:
Turn around time for bug fix or adding feature is more.
Fear of regression issues.
Too much resources and time spent in testing.
Velocity is low.
Maitenance is costly

What is Testability?
Software testability is the degree to which a software artifact supports testing in a given test context.
If the testability of the software artifact is high, then finding faults in the system by means of testing is easier.

Testability Tipes
Use a Dependency Injection framework
Learn at least one in-memory database for integration testing.
Make it a habit to test your code changes only through unit testing.
Automate your tests and make sure your developer cannot check in the code even if one of your tests fails.
Don't try for 100% coverage, aim for covering important functionality.
When you fix a bug, add a unit test first and then fix the bug.
Implement Semantic Monitoring.
Keep your E2E as minimal as possible as simple.
Use browser stack, sause lab or any third party online services for device testing.

Security

Security problems: Always a challenge.
Can make our life as nightmare.
Can make our application go down.
Tough to keep up with the all negative forces.
Your system can be attached at anytime by anyone, by means, intentionally or unintentionally.

What is Security
Security is an ongoing process involving people and practices, and ensures application confidentiality, integrity, and availability.

Security Principles
Don't trust people, any hardware or software.
Audit all users and usage
Implement security in every layer of software from physical and application layer.
Always follow a lease privilige approach. Do not provide even one extra previlege.
Set a time limit for every previlege provided.

Security Principles
Always sanitize input to avoid SQL injection, Cross site scripting and remote file inclusion attacks.
Always use HTTPS and enable SSL certification.
Use encryption wherever applicable.
Always use request limiting, to prevent DDOS attack.
Use the Encryption at REST for available services.
Make sure you keep the connection string in vaults that can only be accessible by your service
Don't try to build a login system, use an already built-in system such as OAuth, AD B2C etc.
Choose managed services rather than unmanaged services provided by the cloud vendor.
Never share production configuraiton to anyone, it should be only in the application settings or in vault

Extensibility

Extensibility Problems
Time for market is low
Defines overall success of the project
Compitition is high, faster to market has become default necessity.
Can make your application unstable.

What is Extensibility
Extensibility is a measure of the ability to extend a system and the level of effort required to implement the extention.
Extensions can be through the addition of new functionality or through modification of existing functionality.
The principle provides for enhanecments, without impairing existing system functions.

Extensibility Tips:
Beware of the antipattern Vendor lock-in.
Start thinking your system in smaller services or microservices. Understand the domain driven design and even driven design.
Automate every manual work that hinders the release process.
Setup Automated rollback mechanism.
Choose buy strategy, when you are presented with buy vs build choice.
Always validate your assumptions, your way of thinking may be the problem that is not allowing the system to extend and scale.
Beware of your knowledge which may stop Extensibility

Observability

Observability Problems
Spend countless time to understand the problem.
Waste lots of time in reproducing the problem.
Need to access the production resource to find the problem.

What is the Observability Observability is a measure of how well internal states of a system can be inferred from knowledge of its external outputs.
Another name is Monitoring.

Observability Components
Data Collection
Data Storage
Visualization
Analytics and Reporting

Observability Tips
Select the right tools, don't select any monitoring tools because amazon or google uses it.
Monitoring is not a Job, so you cannot have manual person sitting and watching the dashboard.
Alert only when humun intervention is needed, remember all ignored alerts in your projects.
Continuous improvement is the key in monitoring systems.
Write logs as much as possible.
Use correlation id in your logs, for time interval. Always set up availability test for all your applications including web App, web API, messaging service etc.

Maintainability

What is Maintainability
Maintainability is the ease with which a product can be maintained in order to correct defects or their cause, prevent unexpected working conditions, make future maintenance easier, or cope with changed environment.

Maintainability Principles
Broken window theory states that visible signs of disorder and misbehavior in an environment encourages further disorder and misbehavior, leading to serious crimes.
Same applies to code base, if you accept one hacky code, your applicaiton become a spaghetti repo.

Maintainability Tips:
Write code for humans.
Choose always a simple way to implement the feature and avoid complexity. Avoid accidental complexity by all means.
Code by thinking first and not by typing.
Try to refactor the code as many times as possible, use the Boy Scout Rule for coding. Leave your code better than you found it.
Have a well defined CI/CD pipelines, so the release to stage is not dependent on any person.
Make sure you have enabled static code analysers such as foxcop, JTest, linter etc.
Never voilate Single Responsibility Principle
Code review can make your code maintainable, just the thought of someone reviewing our code makes us better developer.
Final tip, is the "Sharpen the saw" principle, make it a habit for yourself and also for other developers to learn a new way of coding.
