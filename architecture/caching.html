Caching

Caching Topologies:

  • Single in-memory cache
  • Distributed (Client/Server) cache
  • Replicated (in-process) cache
  • Near cache hibrid

Single in-memory (IMDG/IMDB) cache

IMDG - In Memory Data Grid , IMDB - In memory Database

Products : Apache Ignite, Couchbase, EHCache,NCache, Hazlecast, Memcache, Redis, Pivotal GemFire, Oracle coherance

Distributed (Client/Server) cache

Cache will be located in an external server/service, All the components/services will have access the cache using a client library using propreatory protocal

Products : Apache Ignite, Couchbase, EHCache,NCache, Hazlecast, Memcache, Redis, Pivotal GemFire, Oracle coherance

Replicated (in-process) cache

The data is maitened in the in-memory cache in each instance, but the data is replicated/duplicated across all the instances in the group (using a client library)

Replicating the data is different than mirroring.

Products : Apache Ignite, Hazlecast, Pivotal GemFire, Oracle coherance

Near cache hibrid

This cache is to solve the problems of both in-memory (memory consumption) and distributed cache (latency)

The cache will be maintain in a distributed server but each instance will maintain a smaller version (most frequently used MFU) of the cache in the memory.

Products : Apache Ignite, Hazlecast, Pivotal GemFire, Oracle coherance

Choosing the right caching topology:

For microservices the right toplogies are Distributed or Replicated
